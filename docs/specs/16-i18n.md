ALTERNATIVE: getStaticPaths met slug mapping

In plaats van per-locale folders, gebruik dynamic routing met 
slug-translation lookup:

src/pages/
├── [locale]/
│   └── [...slug].astro

In [...slug].astro:
- Detect locale from URL
- Lookup canonical slug-mapping (NL "voorspellingen" → "predictions")
- Render content based on canonical route + locale

Aanbeveling: dynamic routing approach. Single source of truth 
voor page logic, slug-mapping in config file.

```typescript
export function getLocalizedSlug(
  canonical: string,
  locale: string
): string {
  return slugMappings[canonical]?.[locale] ?? canonical;
}

export function getCanonicalFromLocalized(
  localizedSlug: string,
  locale: string
): string | null {
  for (const [canonical, mapping] of Object.entries(slugMappings)) {
    if (mapping[locale] === localizedSlug) {
      return canonical;
    }
  }
  return null;
}
```

```typescript
import { defaultLocale } from '@/config/site';

export async function getContent(
  pageName: string, 
  locale: string
): Promise<any> {
  try {
    const content = await import(
      `@/content/pages/${pageName}/${locale}.json`
    );
    return content.default;
  } catch (e) {
    // Fallback to default locale
    if (locale !== defaultLocale) {
      console.warn(
        `Translation missing for ${pageName}/${locale}, falling back to ${defaultLocale}`
      );
      const fallback = await import(
        `@/content/pages/${pageName}/${defaultLocale}.json`
      );
      return fallback.default;
    }
    throw e;
  }
}
```

src/lib/i18n.ts (extension):

```typescript
export function getAllLocaleVersions(
  currentPath: string
): Array<{ locale: string; url: string }> {
  const baseUrl = 'https://betsplug.com';
  
  // Detect canonical path from current locale
  const segments = currentPath.split('/').filter(Boolean);
  const detectedLocale = locales.includes(segments[0] as any) 
    ? segments[0] 
    : 'en';
  
  // Build canonical path
  const canonicalSegments = detectedLocale === 'en' 
    ? segments 
    : segments.slice(1);
  
  // Generate URL for each locale
  return locales.map(targetLocale => {
    const localizedSegments = canonicalSegments.map((seg, idx) => {
      // First segment is potentially translated folder name
      if (idx === 0) {
        return getLocalizedSlugFromCanonical(seg, targetLocale);
      }
      // Remaining segments stay as-is (article slugs handled separately)
      return seg;
    });
    
    const path = targetLocale === 'en'
      ? `/${localizedSegments.join('/')}`
      : `/${targetLocale}/${localizedSegments.join('/')}`;
    
    return {
      locale: targetLocale,
      url: `${baseUrl}${path}`,
    };
  });
}
```

═══════════════════════════════════════════════════════
LOCALE DETECTION
═══════════════════════════════════════════════════════

INITIAL LOCALE DETECTION

Priority order:
1. URL prefix (/nl/, /de/, etc.) — explicit choice
2. Cookie 'locale_pref' — previous user choice
3. Accept-Language header — browser preference
4. Default (en)

Implementation in Astro middleware:

src/middleware.ts:

```typescript
import { defineMiddleware } from 'astro:middleware';
import { detectLocale } from '@/lib/i18n';

export const onRequest = defineMiddleware(async (context, next) => {
  const url = new URL(context.request.url);
  const pathLocale = url.pathname.split('/')[1];
  
  // If URL has explicit locale, use it
  if (locales.includes(pathLocale as any)) {
    context.locals.locale = pathLocale;
    return next();
  }
  
  // Otherwise, redirect to user's preferred locale on first visit
  const cookie = context.cookies.get('locale_pref');
  const preferredLocale = cookie?.value 
    ?? detectLocaleFromHeader(
         context.request.headers.get('accept-language')
       )
    ?? 'en';
  
  if (preferredLocale !== 'en' && url.pathname === '/') {
    return context.redirect(`/${preferredLocale}/`);
  }
  
  context.locals.locale = preferredLocale;
  return next();
});
```

LOCALE PREFERENCE PERSISTENCE

When user manually switches locale (via switcher):
- Set cookie 'locale_pref' (1 year)
- Honor cookie on future visits
- Override Accept-Language header

═══════════════════════════════════════════════════════
TRANSLATION WORKFLOW
═══════════════════════════════════════════════════════

PHASE 1: ENGLISH FIRST

All content written in English first:
- Page copy
- Article content  
- FAQ Q&A
- UI labels
- Email templates

Stored in src/content/{type}/{slug}/en.json

PHASE 2: TRANSLATION

Two approaches per content type:

A) Pages with structured copy (homepage, pricing, etc.):
   - Use AI translation (Claude, DeepL Pro) for first pass
   - Native speaker review for tone, cultural fit
   - Legal review for legal pages
   - Stored as separate locale.json files

B) Long-form articles (learn content):
   - AI first-pass translation
   - Native speaker review (full read-through)
   - SEO check (keyword translation correctness)
   - Stored as separate locale.md files

PHASE 3: VERIFICATION

Per locale per page:
- All required fields present (no missing translations)
- Cultural-appropriate examples (Bundesliga voor DE, Eredivisie voor NL)
- Locale-specific compliance language verified
- Native speaker final review

═══════════════════════════════════════════════════════
FALLBACK HANDLING
═══════════════════════════════════════════════════════

When translation missing:

1. Page-level: fallback to English version, log warning
2. Field-level: fallback to English equivalent for that field
3. Never show empty content or placeholder text
4. Console warnings during build for missing translations

Build-time check:

```typescript
// At build time, verify all locales have all required pages
async function verifyTranslations() {
  const requiredPages = [
    'homepage', 'pricing', 'about', 'contact', /* etc */
  ];
  
  for (const page of requiredPages) {
    for (const locale of locales) {
      const path = `src/content/pages/${page}/${locale}.json`;
      if (!fs.existsSync(path)) {
        console.warn(`Missing translation: ${page} for ${locale}`);
      }
    }
  }
}
```
