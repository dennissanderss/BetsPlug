CONTENT STRUCTUUR (per artikel)

Intro paragraph (~150 words):
- Probleem-statement of context
- Wat artikel behandelt
- Wat lezer aan het einde weet

H2 sectie 1: "Wat is [concept]?"
- Definitie (1-2 paragraphs)
- Origin/history (kort, alleen als relevant)
- Visualisatie of diagram inline

H2 sectie 2: "Hoe werkt het?"
- Mechaniek uitleg (3-4 paragraphs)
- Formule of berekening (als applicable)
- Concreet voorbeeld met cijfers

H2 sectie 3: "Waarom matters dit?"
- Praktische toepassing in football predictions
- Waar het sterk is
- Waar het zwak is (limitations)

H2 sectie 4: "Hoe gebruikt BetsPlug dit?"
- Specifieke toepassing in onze engine
- Hoe het integreert met andere modellen
- Link naar /methodology

H2 sectie 5: "Praktische tips" (optioneel, niet voor alle artikelen)
- Hoe lezer dit kan toepassen
- Common mistakes
- Recommended next steps

H2 sectie 6: "Veelgestelde vragen"
- 3-5 FAQ-style Q&A specifiek voor het concept

H2 sectie 7: "Verder lezen"
- 2-3 internal links naar gerelateerde artikelen
- 1-2 external links naar wetenschappelijke bronnen (optioneel)

CONTENT PRINCIPLES

- Schrijf voor intelligente lezer, niet voor expert
- Define terms first time gebruikt
- Use concrete examples, niet alleen abstracte concepten
- Math/formulas: gebruik LaTeX of clean code blocks
- Diagrams: SVG inline waar mogelijk (geen externe images)
- Transparency: erken limitations expliciet
- Internal linking: minimaal 3-5 links per artikel
