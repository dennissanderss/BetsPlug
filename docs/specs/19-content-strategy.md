═══════════════════════════════════════════════════════
CONTENT WRITING PRINCIPLES
═══════════════════════════════════════════════════════

VOICE & TONE (REINFORCEMENT)

Per 00-context-and-goals.md:
- Premium maar toegankelijk
- Transparant, data-driven, professional, honest
- Geen marketing-fluff
- Methodology boven adjectives
- "We" perspectief bij methodology, "you" perspectief bij benefits

WRITING RULES

1. ACTIVE VOICE
   ❌ "Predictions are generated by our engine"
   ✓ "Our engine generates predictions"

2. DIRECT SENTENCES
   ❌ "It's important to understand that AI predictions, while powerful, 
       have limitations that users should be aware of."
   ✓ "AI predictions have limitations. Know them."

3. SPECIFICITY
   ❌ "We use advanced AI"
   ✓ "We combine Elo ratings, Poisson models, and machine learning"

4. NO HEDGING
   ❌ "BetsPlug might help you make better decisions"
   ✓ "BetsPlug provides statistical analysis for your decisions"

5. CONCRETE OVER ABSTRACT
   ❌ "Many advantages over traditional methods"
   ✓ "30+ leagues, 40+ variables per match, locked predictions"

6. NO SUPERLATIVES UNLESS VERIFIED
   ❌ "The best AI football prediction platform"
   ✓ "AI football predictions with public track record"

7. ACKNOWLEDGE LIMITATIONS
   ❌ "Our predictions are highly accurate"
   ✓ "Our predictions are probabilistic, not guarantees"

CONTENT TYPES & GUIDELINES
═══════════════════════════════════════════════════════

PAGE COPY (homepage, pricing, about)

Length: Concise, punchy
Style: Action-oriented, conversion-focused
Structure: Short paragraphs (2-4 sentences max)
CTA-density: 1 primary CTA per 2-3 sections

EDUCATIONAL CONTENT (learn articles, methodology)

Length: 1500-3000 words per article
Style: Technical maar leesbaar, voorbeelden, diagrammen
Structure: Clear H2 hierarchy, TOC, key takeaways summary
Internal linking: 3-5 per article minimum

PRODUCT DESCRIPTIONS (predictions, features)

Length: Short and specific
Style: Functional, benefit-focused
Structure: Feature → benefit → proof point

LEGAL CONTENT

Length: Comprehensive but organized
Style: Clear, formal, no legal-jargon waar te vermijden
Structure: Numbered sections, table of contents

TRANSLATION WORKFLOW
═══════════════════════════════════════════════════════

PHASE 1: ENGLISH MASTER

Process:
1. Write content in English (native quality)
2. Internal review (2 reviewers minimum)
3. SEO check (keyword density, meta tags)
4. Compliance check (gambling terminology, claims)
5. Save to src/content/.../en.json or en.md

Quality gate: cannot proceed to translation until English is final.

PHASE 2: AI FIRST-PASS TRANSLATION

Tool: DeepL Pro (preferred) or Claude/GPT-4

Process per locale:
1. Translate full content
2. Preserve technical terms (Elo, Poisson, xG, Kelly)
3. Adapt cultural references (league examples, etc.)
4. Apply gambling terminology mapping per locale
5. Save as draft to src/content/.../{locale}.json

Translation prompt template:

"Translate the following English marketing content to [LANGUAGE]. 
Context: This is for BetsPlug, an AI football prediction platform.

Critical requirements:
- Preserve technical terms: Elo, Poisson, xG, Kelly Criterion
- Use these terminology replacements:
  [LOCALE-SPECIFIC GAMBLING ALTERNATIVES]
- Maintain professional, data-driven tone
- Don't translate league names (Premier League stays Premier League)
- Format: maintain JSON structure exactly

Source content:
[CONTENT]"

PHASE 3: NATIVE SPEAKER REVIEW

Required for:
- All page copy (homepage, pricing, about, etc.)
- All learn articles
- All legal pages
- Marketing CTAs and headlines

Process:
1. Native speaker reads full translation
2. Checks: tone, idiom, cultural fit, SEO keyword translation
3. Edits inline
4. Approves or rejects with notes

Reviewer brief:
"Review this translation for [LANGUAGE]. Goals:
- Sounds natural to native speaker
- Technical terms accurate
- Cultural references appropriate
- SEO keywords correctly translated
- Compliance language correct (we are NOT a bookmaker)
- Tone matches: professional, data-driven, transparent

Flag any issues. Edit inline where you can improve."

PHASE 4: SEO LOCALIZATION

Per locale review:
- Title tag (60 chars in target language)
- Meta description (155 chars in target language)
- H1 contains primary keyword in target language
- Body has 2-3 keyword mentions naturally
- Internal links use localized anchor text

PHASE 5: QA TESTING

Per locale:
1. Page renders correctly (no broken JSON)
2. All translations present (no English fallbacks)
3. Cultural elements appropriate
4. Date/number/currency formatting correct
5. Links to other locale pages work
6. Schema.org inLanguage attribute correct

CONTENT MAINTENANCE
═══════════════════════════════════════════════════════

UPDATE TRIGGERS

Update content when:
- Pricing changes (immediately)
- Methodology changes (within 1 week)
- Legal requirements change (immediately)
- League rosters/seasons change (start of season)
- New features launch (within 1 week)
- Competitor analysis reveals positioning gap (quarterly)

VERSION CONTROL

All content in Git:
- src/content/ folder tracked
- Commits include locale changes
- Tag major content updates (v1.0, v1.1, etc.)
- Historical versions accessible via Git history

CONTENT FRESHNESS

Per page, "lastModified" date prominently shown for:
- Learn articles
- Legal pages (very prominent)
- Methodology

Hidden but tracked for:
- Marketing pages (homepage, pricing, etc.)
